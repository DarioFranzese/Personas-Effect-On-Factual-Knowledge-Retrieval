# Results Directory

This directory stores raw benchmark runs before answer normalization or metric computation.

## Purpose

Files here are the direct outputs of benchmark execution. They preserve the model responses, prompt context, and run configuration so that later stages can reprocess the same data consistently.

## Structure

- One subdirectory per benchmark: `aime/`, `gpqa/`, `mmlu/`, and `mmlu_pro/`.
- Inside each benchmark folder, results are grouped by model identifier.
- File names encode the reasoning mode, persona setting, and shot mode.

A typical file name looks like:

- `reasoning_true_personas_true_zero-shot.json`
- `reasoning_low_personas_false_few-shot.json`

## Producers

These files are generated by `BenchmarkRunner.run_benchmark()` and by the collection workflow in `code/benchmark_collection.py`.

## Recommended Contents

A raw result file should usually preserve:

- Benchmark name.
- Model identifier.
- Subject or category.
- Question text.
- Ground-truth answer.
- Multiple-choice options when present.
- Raw model output.
- Any metadata needed to reconstruct the run.

## Notes

- Treat this directory as the source of truth for reruns and audits.
- Do not overwrite existing files unless you are intentionally replacing a benchmark run.
- Keep any new documentation in English.
